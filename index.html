Learning Phases

P1
Phase 1 — Foundation
C++ Fundamentals
Before DSA, master the language. Focus on syntax, pointers, memory management, references, and the STL basics. You need to think in C++ before you write algorithms.
⏱ 3 – 4 Weeks
Variables & Types Loops & Conditionals Functions & Scope Pointers & References Arrays & Strings Structs & Classes STL: vector, map, set Memory: new / delete Time & Space Complexity Big O Notation
P2
Phase 2 — Linear Structures
Arrays, Strings, Linked Lists, Stacks & Queues
Build the core linear data structures from scratch. Understand when and why each structure is used. This is the most heavily tested category in coding interviews.
⏱ 5 – 6 Weeks
Static & Dynamic Arrays Two Pointer Technique Sliding Window String Manipulation Singly Linked List Doubly Linked List Circular Linked List Stack (Array & LL) Queue & Deque Priority Queue
P3
Phase 3 — Non-Linear Structures
Trees, Heaps & Hashing
Step into the non-linear world. Trees and graphs are the backbone of advanced problem solving. Master traversals, BST operations, and hash maps deeply.
⏱ 6 – 7 Weeks
Binary Tree Binary Search Tree Tree Traversals (BFS/DFS) AVL Tree Min/Max Heap Hash Table Collision Handling Trie Segment Tree Fenwick Tree (BIT)
P4
Phase 4 — Algorithms
Sorting, Searching, Graphs & Recursion
Now apply structures to algorithms. Learn classic algorithms, graph theory (BFS/DFS/Dijkstra), and divide-and-conquer strategies. This is where you start solving real problems fast.
⏱ 7 – 8 Weeks
Bubble / Selection / Insertion Sort Merge Sort Quick Sort Heap Sort Binary Search Recursion & Backtracking Graph Representation BFS & DFS Dijkstra's Algorithm Bellman-Ford Floyd-Warshall Kruskal & Prim (MST) Topological Sort Union-Find (DSU)
P5
Phase 5 — Advanced
Dynamic Programming, Greedy & Competitive Prep
The final boss. DP is the single most important topic for FAANG interviews and competitive programming. Master patterns: knapsack, LCS, LIS, DP on trees, and bitmask DP.
⏱ 8 – 10 Weeks
Memoization vs Tabulation 1D & 2D DP 0/1 Knapsack Unbounded Knapsack LCS & LIS Matrix Chain Multiplication DP on Trees Bitmask DP Greedy Algorithms Activity Selection Bit Manipulation Math for CP

Pro Tips for Beginners

01
Code before you copyAttempt every problem for at least 20–30 minutes before watching a solution. Struggling builds intuition that watching passively cannot.
02
Master one topic at a timeDon't jump around. Spend a full week on arrays, then move to linked lists. Depth beats breadth for long-term retention.
03
Use Striver's A2Z SheetFollow the takeUforward A2Z DSA Sheet as your primary problem tracker. It's battle-tested for FAANG interviews with 450+ problems in order.
04
Always analyze complexityAfter every solution, ask: "What is the time and space complexity?" This habit separates good programmers from great ones.
05
Enter Codeforces Div. 3 contestsDo a Codeforces Div. 3 contest every week once you finish Phase 2. Timed pressure reveals true understanding vs false familiarity.
06
Revise with spaced repetitionProblems you solve today will fade. Re-attempt solved problems after 3 days, 1 week, and 1 month. Revisit before interviews.
07
Build pattern recognitionMost DSA problems fall into 15–20 patterns (sliding window, two pointers, BFS, DP knapsack…). Learn to recognize the pattern before the problem.
08
Don't ignore STLIn C++, std::vector, std::map, std::priority_queue, std::unordered_map are your best friends. Master the STL — it saves time and avoids bugs in contests and interviews.
09
Consistency over intensity2 hours every day beats 14 hours every weekend. DSA is a marathon. Set a daily streak goal and protect it like a habit, not a chore.
